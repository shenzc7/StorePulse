Data Characteristics & Guarantees

What we collect, how it behaves, and the controls that keep it trustworthy.

Granularity
Daily
One row per location/day. Timezone-normalized to UTC with de-duplication by (event_date, location_id).
Core Measures
Visits, Sales
Visits is the target (non-negative integer). Sales is optional but used for conversion-rate derived signals.
Contextual Drivers
Weather, Promo, Events
Weather icon + temperature, promo type/intensity, paydays, local events, school breaks, open hours.
Volume & Freshness
‚â§10K rows / upload
50MB limit, rolling 365-day window cached, new data arrives daily via Lite/Pro forms or bulk files.
Quality Gates
Type + Range
Schema validation, negative-value rejection, row count enforcement, holiday/weekend flags auto-filled.
Feature Yield
21 ‚Üí 30+
21 standard engineered features; Pro mode can add price elasticity, promo uplift, and seasonality harmonics.

Fields & Properties

Field Type Characteristics Used For
event_date date Unique per day; timezone-normalized; gaps are forward/back-filled where allowed Index, temporal features
visits int ‚â• 0 Target variable; outliers winsorized at P99 Y; lag/rolling/trend signals
sales float ‚â• 0 Optional; derives conversion_rate and revenue_per_visit Demand elasticity, promo impact
promo_type enum None / %off / BOGO / bundle / loyalty Driver encoding, uplift estimation
weather enum + float Icon + temp; missing values imputed from nearest station Weather sensitivity features
local_events text/enum School breaks, concerts, sports; binarized + lead/lag signals Event surge detection
